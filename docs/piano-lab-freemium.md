> Дата: 19.07.2026. Статус: **анализ и предложение владельцу** (реализация —

## 0. TL;DR — три решения

1. **Открываем не «платформу», а привычку.** Лестница: (а) ежемесячный
   e-mail-отчёт — уже собираем согласия чекбоксом в check-up; (б) личный
   кабинет Lab с 1 бесплатным сайтом — **на той же auth, что cab.remarka.biz**
   (magic-link, таблицы `Cab*`, Brevo, i18n — всё построено и обкатано);
   (в) платный ярус. Одна учётка на кабинет клиента и Lab — один вход,
   одна кодовая база, и каждый free-пользователь уже «в прихожей» студии.
2. **Токены не тратим на бесплатное по требованию.** Free-ярус живёт на
   бесплатных API (PSI, CrUX) и rule-based текстах (нулевая себестоимость);
   ИИ — только раз в месяц, батчем, ночью, Haiku, с жёстким глобальным
   капом и деградацией в шаблонные тексты. Себестоимость ИИ на free-юзера —
   **доли цента в месяц**; дорогой ИИ (Sonnet, по требованию, сравнение
   с конкурентами) — только в платном ярусе, где он окупается.
3. **Монетизация — двухмоторная:** подписка Monitor Pro (€24/мес за 3 сайта,
   год = 10 месяцев) плюс главный мотор — **сервисный**: каждый алерт
   «показатель просел» заканчивается кнопкой «чиним мы, prezzo chiuso».
   Free остаётся навсегда (это лидоген, а не убыток). Платное включаем
   через 2–3 месяца после открытия, ранним пользователям — грандфазеринг.

## 1. Что именно открываем: лестница доступа

### 1.1. Почему НЕ «просто снять basic auth»

Monitor-API сегодня без единого guard'а (`GET /api/projects` отдаёт всё) —
открытие требует авторизации и tenant-скоупинга в любом случае. Строить
вторую auth с нуля — расточительно: **в кабинете уже есть** magic-link,
серверные сессии с отзывом, мультитенантность, аудит, письма Brevo,
интерфейс IT/EN/RU. Спека K1 прямо закладывала реюз: «если позже Monitor
захочет тот же вход — переиспользует таблицы `Cab*`».

**Решение:** авторизация Lab = учётка кабинета. `CabUser` получает
membership нового типа (`lab_free` / `lab_pro`), Monitor-роуты проверяют
ту же cookie-сессию. Пользователь, пришедший «бесплатно мониторить сайт»,
логинится там же, где потом будет жить как клиент студии.

### 1.2. Лестница (каждая ступень работает сама по себе)

| Ступень | Что получает пользователь | Что получаем мы | Готовность |
|---|---|---|---|
| **L0 — e-mail-дайджест** | раз в месяц письмо: балл check-up, дельта к прошлому месяцу, 3 приоритета | e-mail + согласие (чекбокс уже в проде, лиды в `sr_lead`) | почти готово: нужен только крон, который гоняет check-up по списку и шлёт письмо |
| **L1 — Lab free** | вход по magic-link, 1 сайт, дашборд: история балла (3 мес.), аптайм ~30 мин, недельный снимок PSI/CrUX, месячный отчёт | привычка «зайти посмотреть», профиль пользователя, поле для апселла | auth реюзом Cab; Monitor-UI урезать до free-скоупа |
| **L2 — Monitor Pro** | §4 | деньги + токены окупаются | после привычки (§4.3) |

L0 — самый быстрый запуск и самый дешёвый тест спроса: если открываемость
писем < 25–30 %, дашборд можно не торопить.

### 1.3. Публичная витрина (обещана после авторизации)

Лендинг на lab.remarka.biz: что измеряем, скрин дашборда, **живой публичный
пример** — дашборд самого remarka.biz (read-only, без auth): «вот как мы
следим за собой». Это честная демка нулевой стоимости и в ДНК «числа
вместо прилагательных».

## 2. Себестоимость free-яруса (почему он почти бесплатный)

| Ресурс | Free-механика | Себестоимость |
|---|---|---|
| Скорость/SEO/a11y/BP | Google PSI API, 1 снимок/нед./сайт | 0 € (квота ключа ~25k/сутки — хватит на тысячи сайтов) |
| Полевые CWV | CrUX API, 1 раз/нед. | 0 € |
| Аптайм | HTTP HEAD каждые 30 мин с нашего box | ~0 (микросекунды CPU) |
| Краул Chromium | **не для free** (только клиентские сайты) | 0 — главная защита 4 ГБ box |
| Письма | Brevo free 300/день → месячные отчёты волнами по дням | 0 € до ~9000 юзеров |
| Тексты вердиктов | rule-based (как data-verdict-* в виджетах) | 0 токенов |
| ИИ-комментарий | 1 абзац/мес., Haiku, батчем ночью | см. §3 |

Хостинг: free-ярус не добавляет контейнеров. Триггер рескейла CX23→CX33
(8 ГБ, 2-минутный ребут) — по `docker stats`, а не заранее.

## 3. Экономика токенов: семь правил

Действующая база (ai-tools.php): кэш 24ч по домену, 3/день/IP/инструмент,
два дневных капа (smart=Sonnet, fast=Haiku), режим «manutenzione» без
ключа, бюджет ≤€20/мес. Для Lab расширяем теми же принципами:

1. **ИИ никогда не по требованию на free.** Только плановая генерация:
   1 короткий комментарий к месячному отчёту. Пользователь не может
   «накликать» расход.
2. **Haiku для free, Sonnet только там, где платят.** Уже так в tools —
   сохранить границу в Monitor.
3. **Rule-based прежде ИИ.** Дельты, светофоры, приоритеты — детерминированные
   тексты (0 токенов). ИИ пишет только «прочтение» — то, что правилами
   не выразить.
4. **Batch API + prompt caching.** Ночная пачка месячных комментариев через
   Batch (−50 % к цене) с общим системным промптом в кэше. Порядок величин:
   комментарий ≈ 2k токенов входа + 0,5k выхода на Haiku — **меньше цента**;
   даже 1000 free-юзеров — единицы евро в месяц.
5. **Глобальный месячный кап с деградацией.** Кап исчерпан → отчёты уходят
   с rule-based текстом вместо ИИ-абзаца, сервис не ломается (паттерн
   «manutenzione» уже отработан).
6. **Кэш по домену и по месяцу.** Один домен = один комментарий в месяц,
   сколько бы людей его ни смотрели.
7. **Дорогие ИИ-фичи = платные фичи.** Сравнение с конкурентами (запрос
   Şevket с Product Hunt), «сайт глазами ИИ» по всем страницам, план
   работ — только в Pro: токены тратятся там, где есть выручка.

## 4. Монетизация

### 4.1. Ярусы

| | **Lab Free** | **Monitor Pro** (~€24/мес или €240/год) | **Agency** (позже, ~€79/мес) |
|---|---|---|---|
| Сайтов | 1 | 3 | 20 |
| Снимок метрик | раз в неделю | ежедневно | ежедневно |
| Аптайм | ~30 мин | 1–5 мин + мгновенный алерт | 1–5 мин |
| История | 3 мес. | 12+ мес. | 12+ мес. |
| Отчёт | месячный e-mail, rule-based + 1 ИИ-абзац | месячный PDF с ИИ-разбором (Sonnet) | white-label PDF под брендом агентства |
| Сравнение с конкурентами | — | 2 соперника | 5 |
| Краул сайта (Chromium) | — | да, по расписанию | да |
| Человек смотрит | — | да, «se un valore peggiora lo vediamo noi» | — |

Клиенты студии с поддержкой получают Pro включённым (уже обещано на сайте).
Цена — «ниже среднего с гарантиями»: дешевле Semrush на порядок, дороже
голого UptimeRobot, потому что продаём не графики, а «il tecnico che
guarda». Публичная цена, годовая = 10 месяцев, prezzo chiuso.

### 4.2. Главный мотор — сервисный

Monitor — это машина квалифицированных лидов: каждый алерт и каждый
красный светофор заканчивается CTA «чиним мы: prezzo chiuso, 24 ore».
Один конвертированный restyling (€2 900+) окупает годы хостинга free-яруса.
Поэтому free — не «расход на маркетинг», а витрина с горячими лидами,
и его не надо душить лимитами до бесполезности.

### 4.3. Последовательность («когда привыкнут»)

- **Фаза 0 (сейчас):** L0-дайджест + L1 free, платного нет вообще. Копим
  когорту, смотрим §5-метрики. Приглашение первой волной — по базе
  согласий из check-up и по community Product Hunt.
- **Фаза 1 (через 2–3 мес., если ≥100–200 активных):** включаем Pro.
  Ранним пользователям — грандфазеринг лимитов + скидка запуска (в ДНК
  «prezzo lancio primi 5»). Free не урезаем задним числом — доверие дороже.
- **Фаза 2:** Agency-ярус + сравнение с конкурентами + white-label — по
  спросу из фидбек-виджета (он уже собирает пожелания).

## 5. Метрики и триггеры решений

| Метрика | Порог действия |
|---|---|
| Открываемость L0-писем | <25 % — чиним письмо, не строим дашборд |
| Активация L1 (добавил сайт и вернулся на неделе 2) | <20 % — упрощаем onboarding |
| Free→Pro конверсия | цель 2–5 %; <1 % через 3 мес. после запуска Pro — пересобрать оффер |
| Токены/юзер/мес | > прогнозного ×3 — ищем дыру в капах |
| `docker stats` worker/api | стабильно >85 % — рескейл CX33 |
| Заявки «чиним мы» из алертов | главный KPI всей затеи |

## 6. Безопасность, GDPR, юридическое

- Tenant-скоупинг в Monitor-API **до** снятия basic auth — нулевой
  компромисс (паттерн `requireSession`/`assertProjectAccess` из K1).
- Privacy policy: раздел про Lab-учётки (тот же, что для кабинета, + цель
  «мониторинг указанных вами сайтов»); удаление — каскадом (уже в схеме).
- ToS для free: fair use, без SLA, право лимитировать; мониторить можно
  только сайты, которыми пользователь вправе распоряжаться (галочка).
- Анти-абьюз: лимит доменов на учётку, запрет приватных IP (SSRF-паттерн
  уже есть), одноразовые e-mail-домены — в чёрный список при появлении.
