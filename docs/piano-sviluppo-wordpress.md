# Studio Remarka — план разработки сайта на WordPress (remarka.biz)

> Рабочий документ. Основан на: концепции развития (`concept-sviluppo-studio.md`),
> утверждённой дизайн-системе (реализована в Astro-прототипе в этом репозитории)
> и анализе текущего сайта remarka.biz (WordPress 7.0, родительская тема **Prespa**,
> дочерняя тема-скин **Prespa SaaS**, плагин nasio-blocks, контент на Gutenberg).

---

## 1. Архитектурное решение

### 1.1. Что показал анализ remarka.biz

- **Prespa** — классическая тема (не FSE): шапка/подвал рендерятся PHP-шаблонами
  и настраиваются через Customizer; контент страниц собирается из Gutenberg-блоков.
- **Prespa SaaS** — не отдельная тема, а дочерняя тема-скин с набором блочных
  паттернов (`ps-patterns`) и CSS-анимациями.
- Система анимаций Prespa: элементу задаётся класс `p-animation-*`
  (`zoom-in`, `text-transform`, `text-moveUp`, `down-up`, `move-sideways`),
  JS родительской темы добавляет `.animated` при появлении в viewport —
  CSS доигрывает переход. Контракт простой и переиспользуемый.
- Текущий сайт — «Remarka — Translation Platform for Business» (ru-RU) —
  сайт переводческого направления группы.

### 1.2. Решение: собственная дочерняя тема `remarka-studio`

Вместо правок Prespa SaaS создаётся **новая дочерняя тема** `remarka-studio`
(Template: prespa). Причины:

1. **Обратимость** — Prespa SaaS остаётся нетронутой, переключение тем = откат.
2. **Обновляемость** — родительская Prespa обновляется штатно; вся кастомизация
   изолирована в дочерней теме.
3. **Скорость** — свой CSS ~20 КБ вместо каскада надстроек поверх чужого скина;
   без новых JS-библиотек (собственный скрипт ~4 КБ).
4. **Удобство редактирования** — весь контент собирается из Gutenberg-паттернов
   на core-блоках: редактор меняет тексты/цифры, дизайн держат классы и theme.json.

### 1.3. Распределение ответственности

| Слой | Кто отвечает |
|---|---|
| Каркас страницы, шапка, подвал, меню | родительская Prespa + CSS-переопределения дочерней темы |
| Дизайн-токены (цвета, шрифты, размеры, отступы) | `theme.json` дочерней темы → пресеты Gutenberg |
| Компонентные стили (barra, карточки, таблицы, тёмные секции) | `assets/css/remarka.css` |
| Скролл-анимации Prespa (`p-animation-*`) | JS родительской темы (+ фолбэк-observer в нашем JS) |
| Barra dei cento, счётчики, каскадный reveal | `assets/js/remarka.js` (порт из Astro-прототипа) |
| Контент секций | Gutenberg-паттерны (категория «Remarka») |
| Формы | плагин форм (рекомендация: Fluent Forms / CF7) + стили в remarka.css |

### 1.4. Стратегия скорости (PageSpeed 90+ на WordPress)

Бюджет производительности: **LCP < 2,0 s (mobile), CLS < 0,05, JS дочерней темы < 5 КБ**.

1. Шрифты — самохостинг woff2 в теме (`font-display: swap`, preload для двух
   критичных начертаний). До загрузки файлов владельцем тема автоматически
   откатывается на CDN (Fontshare/Google) — переключение по `file_exists`.
2. Отключение эмодзи-скрипта WP, `wp-embed` и dashicons для гостей.
3. Никаких слайдеров/каруселей в дизайне → Swiper из nasio-blocks на страницах
   не грузится (проверить условную загрузку плагина; при необходимости dequeue).
4. Изображения: WebP/AVIF, `loading="lazy"` (WP штатно), явные размеры (анти-CLS).
5. Кэш-плагин + object cache на хостинге (LiteSpeed Cache / WP Rocket — по хостингу).
6. Критичный CSS невелик (один файл ~20 КБ) — инлайн не требуется.
7. Ежемесячный замер PSI (медиана 3 прогонов) — то, что продаём, измеряем у себя.

## 2. Контент: правила «очеловеченного» наполнения

Тон уже определён брендом: **числа вместо прилагательных**. Правила редактуры
для всех страниц (итальянский):

1. **Каждое обещание — с числом или документом.** Не «siamo veloci», а
   «PageSpeed 90+, scritto nell'art. 4 del contratto».
2. **Пишем, как компетентный инженер объясняет владельцу бизнеса за кофе.**
   Короткие фразы. Активный залог. Без канцелярита и без англицизмов, где есть
   итальянское слово (кроме устоявшихся: PageSpeed, e-commerce, CMS).
3. **Обращение на «voi»** — последовательно на всём сайте (уже так в прототипе).
4. **Запрещённые слова** (из tone of voice): rivoluzionario, leader di mercato,
   soluzione a 360°, qualità senza compromessi, innovazione al tuo servizio,
   гарантии позиций в Google.
5. **Каждая страница отвечает на один вопрос клиента** и заканчивается одним
   действием (CTA). Не два CTA равного веса.
6. **Микротексты не из шаблона**: подписи к полям, сообщения об отправке,
   пустые состояния — переписываются вручную (это и отличает премиум).
7. **Правило проверки**: прочитать вслух; если фраза звучит как «сайт агентства» —
   переписать. Если фразу нельзя доказать — удалить или добавить факт.
8. Плейсхолдеров и вымышленных данных на публичном сайте не остаётся
   (P.IVA, телефоны, кейсы, отзывы — только реальные).

Источник текстов: копирайт Astro-прототипа (финального качества по хендоффу) →
редактура носителем → перенос в паттерны.

хелперы блочной разметки Gutenberg (JSON через `json.dumps`, без ручного
экранирования), `generate_pages.py` собирает из них полные страницы. Чтобы
изменить контент — правится `data.py`, затем `python3 generate_pages.py`
перегенерирует все 26 файлов за секунду. Это осознанный выбор вместо ручной
вёрстки 26 страниц блочной разметкой — надёжнее на этом масштабе и позволяет
править контент в одном месте.

## 5. Редакционный workflow в Gutenberg

1. Все секции вставляются из **Паттерны → Remarka** (11 шт., ниже — состав).
2. Редактор меняет **только тексты, числа и ссылки**; классы и структуру не трогает
   (паттерны собраны из core-блоков, стили привязаны к классам).
3. Цвета и шрифты выбираются только из пресетов темы (палитра «Remarka» в
   theme.json) — чужих цветов в интерфейсе редактора нет.
4. Анимации: группе-обёртке добавляется CSS-класс `p-animation-zoom-in` /
   `p-animation-text-moveUp` (движок Prespa) или `sr-reveal` (наш, с каскадом
   `sr-cascade`); barra — готовые HTML-блоки внутри паттернов.
5. Новые страницы = копия существующей + замена паттернов. Ничего не верстается
   «с нуля» в редакторе.

## 6. Критерии готовности

- Lighthouse mobile ≥ 90 (Home + Servizio + Prezzi), CLS < 0,05.
- Нет плейсхолдеров и вымышленных данных; интерфейс полностью на итальянском.
- Баннер cookie: Rifiuta = Accetta по весу; выбор персистентен.
- `prefers-reduced-motion` отключает все анимации (Prespa + наши).
- Формы работают, письма доходят, события аналитики стреляют.
- Sitemap/robots/canonical/schema валидны; редиректы со старых URL — 301.
- Тема обновляема: родительская Prespa обновляется без потери кастомизации.

## 7. Риски и открытые вопросы

1. **Лицензии шрифтов**: Clash Display / General Sans (Fontshare) — проверить
   право самохостинга для коммерческого сайта; альтернатива — замена на
   визуально близкие открытые (например, Space Grotesk + Inter) — решение владельца.
2. **Prespa обновления**: контракт `.p-animation-* → .animated` может измениться;
   в нашем JS есть фолбэк-observer, риск низкий.
3. **remarka.biz сегодня — сайт переводческой платформы.** Куда он переезжает?
   Варианты: поддомен (translate.remarka.biz), раздел, или студия живёт на другом
   домене. Нужно решение владельца до фазы C (влияет на редиректы и SEO).
4. **Хостинг**: тариф/панель неизвестны — от этого зависит выбор кэш-плагина
   и возможность object cache.
