# Studio Remarka — правила для Claude в этом репозитории

## Отчёты владельцу о деплое (требование владельца, 15.07.2026)

При КАЖДОМ сообщении владельцу о готовых изменениях приводить **полный
копипаст-готовый блок команд для сервера** — от бэкапа до очистки кэша.
НЕ писать сокращений вида «это контент → полный цикл с REMARKA_FORCE=1»
или «сценарий 2 из шпаргалки» — владелец вставляет команды в терминал
как есть, каждый раз.

Шаблоны блоков (актуальные пути и оговорки) — `docs/deploy-cheatsheet.md`:
- изменился контент страниц (patterns/**, deploy-import.php, новые
  страницы/статьи) → блок «Полный цикл»;
- изменился только код темы (assets/, functions.php, inc/, lib/) →
  блок «Синк темы»;
- один файл из assets/ → блок «Точечный файл» (только по хэшу коммита).

Если менялись и контент, и код — давать один блок «Полный цикл» (он
покрывает всё). Ключевые оговорки не опускать: `REMARKA_FORCE=1` — это
переменная окружения перед `wp eval-file` (флаг `--force` не работает),
точечный `curl` — только по хэшу коммита (CDN кэширует ветку).

## Прочие обязательные правила

- Правила всех агентских этапов: `docs/piano-implementazione-fase-A.md` §0
  (атомарные коммиты без ID моделей, протокол верификации, грабли:
  `main()` в generate_pages.py запрещён — только точечные вызовы с
  git diff; `translate_pages.py ru` запрещён навсегда).
- Требования владельца ко всему новому контенту: SEO (фокус-ключи из
  docs/seo-meta.md), mobile-first (проверка от 390px), E-E-A-T (источники,
  честные оговорки, без выдуманных фактов/экспертов/отзывов).
- Блог: правила и очередь — docs/piano-blog.md (ежедневный конвейер).
- Деплой-раннбук с объяснениями: docs/deploy-ssh.md.
