# Studio Remarka — правила для Claude в этом репозитории

## Отчёты владельцу о деплое (требование владельца, 15.07.2026)

При КАЖДОМ сообщении владельцу о готовых изменениях приводить **полный
копипаст-готовый блок команд для сервера** — от бэкапа до очистки кэша.
НЕ писать сокращений вида «это контент → полный цикл с REMARKA_FORCE=1»
или «сценарий 2 из шпаргалки» — владелец вставляет команды в терминал
как есть, каждый раз.

Шаблоны блоков (актуальные пути и оговорки) — `docs/deploy-cheatsheet.md`:
- изменился контент страниц (patterns/**, deploy-import.php, новые
  страницы/статьи) → блок «Полный цикл»;
- изменился только код темы (assets/, functions.php, inc/, lib/) →
  блок «Синк темы»;
- один файл из assets/ → блок «Точечный файл» (только по хэшу коммита).

Если менялись и контент, и код — давать один блок «Полный цикл» (он
покрывает всё). Ключевые оговорки не опускать: `REMARKA_FORCE=1` — это
переменная окружения перед `wp eval-file` (флаг `--force` не работает),
точечный `curl` — только по хэшу коммита (CDN кэширует ветку).

## Правила вёрстки (частые ошибки — не повторять)

- **Сетки карточек без «сироты».** Не оставлять одну карточку одну на
  последней строке. Для N карточек выбирать раскладку, где последний ряд
  НЕ из одной: 5 → **3+2** (не 4+1), 4 → 2+2 или 4-в-ряд, 7 → 4+3/3+4 и т.п.
  На узких экранах — одна колонка. (Требование владельца 19.07.2026:
  «некрасиво, когда 1 карточка переносится на следующую строку».)
- **Равная высота карточек в ряду** — карточки одного ряда выравниваются по
  высоте (grid align stretch / равные min-height), без рваных низов.
- Проверять раскладку на 390px (mobile-first) и на десктопе перед сдачей.
- **Надписи должны ВМЕЩАТЬСЯ — проверять визуально, а не только по overflow
  страницы.** Требование владельца 22.07.2026: «2 settimane» в крупном моно
  `.sr-stat__num` переносилось внутри колонки («settiman/e»), при этом
  горизонтального overflow страницы не было — проверка `scrollWidth==viewport`
  это НЕ ловит. Перед сдачей рендерить страницу (Playwright) и проверять
  каждый ключевой элемент на перенос ВНУТРИ бокса: `getClientRects().length>1`
  и `scrollWidth>clientWidth` для `.sr-stat__num`, кнопок, пилюль, eyebrow.
  Крупные цифры — только короткие значения; слово-единицу выносить в
  `.sr-stat__unit` (маленький токен), значение-слово без цифры — класс
  `.sr-stat__num--sm`. Скрипт-аудит: scratchpad `audit-labels.mjs`.
- **То же правило — для текста ВНУТРИ SVG-иллюстраций блога.** 23.07.2026:
  в `immagini-velocita-schema.svg` (батч 9) строка текста вылезала за свою
  фон-плашку (`<rect>`) на ~22px — тоже невидимо для проверки overflow
  страницы, т.к. сам SVG вписывается в контейнер (масштабируется целиком).
  Перед сдачей новых SVG для блога — гонять bbox-аудит текста против
  viewBox И против соседнего `<rect>` в той же группе (`getBBox()` в
  безголовом браузере), не только «на глаз» по скриншоту-монтажу. Скрипт:
  scratchpad `svg-overflow-check.mjs`.
